=============================================================
                 PERPETUAL PROGRESSION
From Revising Java to Building a Banking System in 22 Days
=============================================================

-------------------------------------------------------------
                        OVERVIEW
-------------------------------------------------------------

I would like to express my gratitude to Ms. Sumita Arora, whose
book ISC class 11 JAVA, played an important role in helping me 
build a strong foundation in Java programming. Many of the concepts 
revised before starting this project were learned through her work,
which made this project possible.

This project was started after completing my revision of Java
File Handling(the final topic covered in the book) on 3 June 2026. My 
original goal was to apply the concepts I had revised through my ISC 
Class 11 Java book in a practical project rather than simply solving 
textbook problems.

The concepts I intended to revisit included:

• Classes and Objects
• Constructors
• Method Overloading
• Arrays
• Loops and Conditional Statements
• File Handling
• Basic Exception Handling
• Basic Data Structures and Program Design

Initially, my goal was straightforward: create a banking system
that could store account information using files instead of
temporary variables. At the time, account creation, deposits, 
withdrawals, and data persistence were the primary objectives and 
I never expected the application to grow beyond basic account 
creation, deposits, and withdrawals.

As each feature was completed, I found myself asking,
"What if the system could do this as well?"

With every feature completed, new possibilities emerged.
Each solved problem revealed another challenge worth
exploring. What began as a basic file-handling exercise evolved
into a much larger system featuring transaction history,
authentication, annual interest calculations, a custom date
calculation engine, financial-year tracking, SFT monitoring,
account suspension workflows, current-account penalties, grace
period management, and extensive state management.

The thrill of solving one problem after another became the
driving force behind the project. Each completed feature led
to a more challenging one, pushing the project far beyond its
original scope and teaching me lessons that extended well
beyond Java syntax. I learned that integration is often harder than
building individual features, persistence is more challenging
than simple variables, and software projects tend to grow far
beyond their original scope.

One thing I discovered during this project was how addictive
problem solving can be. Whenever I completed a feature, my mind
immediately started thinking about what could be added next.
Personally, I found this process more exciting than simply
learning new concepts. Even while revising Java, I often caught
myself thinking about the next improvement I wanted to make to
the project. It felt like a child being shown a chocolate and
being unable to think about anything else.
After completing the banking portion of the project, I realised
that I had finally reached a limitation that could not be solved
with the concepts I currently knew. Features such as loan
accounts, insurance systems, and multiple customer accounts all
require proper multi-user handling.
At the moment, the lack of multi-user support remains the
biggest weakness of this project and the next challenge I hope
to solve in the future.

This project is not intended to represent real banking
software. Instead, it serves as a record of my learning
journey and my attempt to apply programming concepts to a
growing, interconnected system.

In many ways, the project itself reflects its title:
Perpetual Progression.

-------------------------------------------------------------
                        HOW TO RUN
-------------------------------------------------------------

1. Download or clone the repository.
2. Open the project in IntelliJ IDEA, BlueJ, or any Java IDE.
3. Compile all Java source files.
4. Run the Bank class.
5. Follow the on-screen menu instructions.

-------------------------------------------------------------
                    KEY LESSONS LEARNED
-------------------------------------------------------------

• Persistence is harder than variables.
• Integration is harder than feature creation.
• Date calculations affect multiple systems.
• State management becomes increasingly important as
  projects grow.
• Refactoring requires careful testing.
• Software projects rarely stay the size originally planned.
